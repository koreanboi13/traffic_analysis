# Анализ безопасности системы

## 1. Аутентификация и авторизация

| Аспект | Реализация | Уровень защиты |
|--------|-----------|---------------|
| Хранение паролей | bcrypt-хэширование (`golang.org/x/crypto/bcrypt`) | Высокий |
| Сессии | JWT-токены (HS256) с настраиваемым TTL (по умолчанию 24ч) | Средний |
| Ролевая модель | RBAC: Admin (полный доступ), Analyst (только чтение) | Средний |
| Передача токена | Заголовок `Authorization: Bearer <token>` | Средний |

### Рекомендации

- Сменить `jwt_secret` с дефолтного `dev-secret-change-me` на криптографически стойкий ключ (минимум 256 бит) в production.
- Рассмотреть переход на RS256 (асимметричные ключи) для production-сценариев.
- Добавить механизм отзыва (revocation) JWT-токенов.

## 2. Защита от инъекций

| Аспект | Реализация | Уровень защиты |
|--------|-----------|---------------|
| SQL-инъекции в WAF | Сигнатурный детект (6 regex-правил) + эвристический анализ (2 правила) | Средний |
| XSS в WAF | Сигнатурный детект (6 regex-правил) + эвристический анализ (1 правило) | Средний |
| Regex engine | stdlib `regexp` (RE2) — гарантия линейного времени выполнения | Высокий |
| SQL-инъекции в инфраструктуре | Валидация имени БД ClickHouse regex-паттерном `^[a-zA-Z_][a-zA-Z0-9_]*$` | Высокий |
| Размер тела запроса | `http.MaxBytesReader` — ограничение 1 МБ для анализа, 1 МБ для API-запросов | Средний |

### Рекомендации

- Расширить набор правил детекции для покрытия дополнительных векторов атак (XXE, SSRF, path traversal).
- Добавить механизм обновления правил из внешних источников (OWASP CRS).

## 3. Сетевая безопасность

| Аспект | Реализация | Уровень защиты |
|--------|-----------|---------------|
| CORS | Ограничение origin: `localhost:5173`, `localhost:3000` | Средний |
| TLS | Поддерживается (TLS termination на прокси) | Средний |
| Порты | Proxy :8080, Admin API :8090, Metrics :9090 — разделение по назначению | Средний |
| ClickHouse (prod) | Доступен только через внутреннюю сеть Docker (`expose`, не `ports`) | Высокий |

### Рекомендации

- В production настроить TLS-сертификаты для всех публичных эндпоинтов.
- Ограничить доступ к Admin API и метрикам через firewall или VPN.
- Настроить CORS с конкретными production-доменами вместо localhost.

## 4. Защита данных

| Аспект | Реализация | Уровень защиты |
|--------|-----------|---------------|
| Хранение событий | ClickHouse с батч-записью, данные хранятся в исходном и нормализованном виде | Средний |
| Конфигурация | YAML-файл, секреты через переменные окружения в prod | Средний |
| Логирование | Structured logging (zap, JSON-формат), без логирования секретов | Средний |

### Рекомендации

- Настроить ротацию и TTL для событий в ClickHouse для соблюдения политик хранения данных.
- Рассмотреть шифрование чувствительных полей в событиях (тело запроса, cookies).
- Не хранить полное тело запроса при наличии конфиденциальных данных — использовать `max_body_size` и маскирование.

## 5. Конфигурация и секреты

| Аспект | Реализация | Уровень защиты |
|--------|-----------|---------------|
| JWT-секрет | Конфигурация в `config.yaml` / переменные окружения | Низкий (dev) / Средний (prod) |
| Пароли БД | Переменные окружения в docker-compose.prod.yml | Средний |
| Горячая перезагрузка | Viper `WatchConfig()` для конфигурации | Средний |

### Рекомендации

- Использовать Docker Secrets или HashiCorp Vault для управления секретами в production.
- Не хранить секреты в `config.yaml` — выносить в переменные окружения.

## 6. Allowlist и исключения

Система поддерживает allowlist для исключения из анализа:
- По IP-адресам (включая `127.0.0.1`, `::1` для локальной разработки).
- По путям (например, `/healthz` для health check).

### Рекомендации

- Регулярно пересматривать allowlist для предотвращения обхода WAF через доверенные пути.
- Логировать запросы, прошедшие через allowlist, для аудита.

## 7. Общая оценка

| Категория | Оценка |
|-----------|--------|
| Аутентификация | Удовлетворительно |
| Детекция атак | Удовлетворительно |
| Сетевая безопасность | Удовлетворительно (dev) / Требует настройки (prod) |
| Управление секретами | Требует улучшения для production |
| Логирование и аудит | Удовлетворительно |

Система обеспечивает базовый уровень безопасности, достаточный для учебного проекта. Для production-использования необходимо реализовать рекомендации, перечисленные в каждом разделе.
